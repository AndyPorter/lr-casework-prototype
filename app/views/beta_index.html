Digital Mortgage Processing BETA Prototypes

This is an index of all versions of prototypes produced in the BETA phase of Digital Mortgage Processing.

Version 1 - 21st December 2015

Change log
  • Initial version
  • Iterated from final ALPHA version here

Version 2 - 9th February 2016

Change log
  • Added service start page
  • Removed table of applications. All applications need to be processed so no need to choose from a list, just get one to start processing.
  • Where there are no related applications, process flow goes straight to draft register check.
  • Added page heading where single question is asked. Are these related? Is this register correct? etc.
  • Application currently being processed is always in sub-heading of page.
  • Related applications page now compares two applications side by side, highlighting the pertenant details.
  • Negative action on both question pages now asks the user to describe why in free text, no template.
  • Removed application details from right side of draft register page.
  • Removed infill value highlighting from draft register entries.
  • Removed manual mark off step from flow. Thought this was best confirmed on negative completion of process.
  • Positive completion of application processing shows confirmation that the application on title will be automatically processed.
  • Negative completion of application processing shows confirmation that the application on title will not be automatically processed and will be available for manual processing in existing systems.
  • At the end of each application processing, a next application button appears, until there are no more to process.

Version 3 - Work in Progress

Change log
  • Removed completion count and percentage. This will appear on a dashboard style view for managers, not all users.
  • Always show related applications step even if no related applications. Keep the experience for each application process flow consistent.
  • Add "What happens next" to confirmation screens (postive and negative).
  • Related applications layout change to a tabular format. More easy to scan and compare information.
  • Removed conveyancer from related applications.
  • Changed confirmation screen text for final application, making it clear there are no more to do.
