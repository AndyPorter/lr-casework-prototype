Change log

The following changes are based on the UX design team peer reviews and usability critique of version 1. Research findings from the Alpha were taken in to account, as there was no direct research / testing evidence from the beta to use.

  • Added service start page. Explained what service is and is not about, so users know what to expect.
  • Removed table of applications. All applications need to be processed so no need to choose from a list, just get the next one to do.
  • Where there are no related applications, process flow goes straight to draft register check. Don't ask the user to complete redundant checks.
  • Added page heading where single question is asked. Are these related? Is this register correct? etc. User should be clear on what they need to do, answer the question heading.
  • Application currently being processed is always in sub-heading of page. Give user context to the question being asked.
  • Related applications page now compares two applications side by side, highlighting the pertenant details to the user.
  • Negative action on both question pages now asks the user to describe why in free text.
  • Removed application details from right side of draft register page. Discourage users from existing mindset of checking everything twice, just focus on one task per page.
  • Removed infill value highlighting from draft register entries. Users aren't responsible for drafting, so it's not just the infil values that are important.
  • Removed manual mark off step from flow. This is best confirmed on negative completion of process, clearer to users they've reach an end and why.
  • Positive completion of application processing shows confirmation that the application on title will be automatically processed.
  • Negative completion of application processing shows confirmation that the application on title will not be automatically processed and will be available for manual processing in existing systems.
  • At the end of each application processing flow, a next application button appears, until there are no more to process, then the user can return to the service home page.

Change log

The following changes were made based on user research and testing conducted in Weymouth office on 9th and 10th of February 2016. Summary of evidence here.

  • Removed completion count and percentage. This will appear on a dashboard style view for managers, not all users.
  • Always show related applications step even if no related applications. Keep the users' experience for each application process flow consistent.
  • Add "What happens next" to confirmation screens (postive and negative). Make it clear to the user what has happened and what will happen now. Give them cause to read this page for useful information.
  • Related applications layout change to a tabular format. More easy to visually scan and compare information.
  • Removed conveyancer from related applications. Charge proetects the interests of the lender, so differences in conveyancer are acceptable.
  • Changed confirmation screen text for final application, making it clear there are no more to do. When there are no more to do user is told they have finished.
  • Added "Step x of y" to processing pages to give user a clear indication of progress made.

General

Outstanding Issues
  • Consistent internal service style inc. header, typography, layout, colour palette, UI components. Work in progress.
  • Users generally uncomfortable with why they are having to do this. Not a recognisable problem as does not fit current process / culture / mental models.
  • Not a USER need to perform this service. Stakeholder / business needs.
